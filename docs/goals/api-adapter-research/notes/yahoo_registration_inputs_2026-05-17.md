# Yahoo!デベロッパー登録 入力方針メモ

Date: 2026-05-17

## 結論

Yahoo!デベロッパー登録の「アプリケーションの説明」は、公式情報上は管理者にのみ表示され、ユーザーには表示されない。
ただし、審査・後続拡張・誤解防止を考えると、「物販自動化」だけではなく、まずは読み取り用途であることが分かる説明にする。

## 推奨入力

アプリケーション名:

```text
輝煌堂 物販API検証
```

アプリケーションの説明:

```text
輝煌堂 物販API検証は、物販業務における価格調査・商品情報確認を目的としたAPI検証用ツールです。
Yahoo!ショッピングAPIから取得した商品情報を、社内検証および仕入れ判断の補助に利用します。
```

## 理由

- 「物販自動化」だけだと、注文・出品・価格変更まで自動実行する印象が出る。
- 現段階の安全な目的は、商品検索・価格調査・仕入れ判断補助の読み取りAPI。
- サイトURL、プライバシーポリシーURL、利用規約URLは利用者同意画面に表示されるため、説明文よりも公開ページ側の整備が重要。
- 属性取得APIで氏名・メールアドレスなどを取得する場合、プライバシーポリシーURLと利用規約URLがないと情報取得できない。

## 次回やること

1. 右側Windowにこのタスクボードを表示して、Mitaさんに見えている確認を取る。
2. 輝煌堂 物販API検証用の簡易公開ページを作るか確認する。
3. ページを作る場合、以下を用意する。
   - トップページ
   - プライバシーポリシー
   - 利用規約
   - Yahoo! APIクレジット表示
4. 公開URL作成は外部公開に当たるため、実行直前にMitaさんへ確認する。
